# Henry Academy class
## Practical Machine Learning and AI Engineering 


* Instructor : [Carlos Daniel Jimenez](https://carlosdanieljimenez.com/)
* Github: https://github.com/carlosjimenez88M


The focus of this repository is to centralize while democratizing the classes corresponding to AI Engineering and, along the way, Applied Machine Learning, together with reinforcing material that includes Python Programming, Docker basics, and FastAPI.

One of the important details to keep in mind regarding these courses is that the focus is on thinking about solutions in a Software format; therefore, managing a modular structure across the repositories will be dominant in each of the lessons (allowing us to also cover organization and GitOps along the way).


Based on the above, this course focuses on explaining how to build systems that learn and adapt to real-world applications; however, starting from the conceptualization phase, the goal is to explore creativity and problem comprehension before focusing on the implementation details of libraries and other tools.

## Course Requirements

Installations will be handled via **uv**; therefore, the following configuration is recommended for a smooth class experience:

* Python version 3.12
* LangChain
* langchain-core
* langchain-openai
* langchain-community
* langchain-text-splitters
* langgraph-checkpoint
* LangGraph
* NumPy
* Matplotlib
* Seaborn
* Scikit-learn
* FastAPI
* ChromaDB
* OpenAI

### Schedule

**AI Engineering**

#### Course 1 — Prompt Engineering with LangChain & LangGraph

Each class is a self-contained `uv` project, reproducible with `langgraph dev`.

| Class | Objective | Link |
| --- | --- | --- |
| Class 1 — Fundamentals | System vs user prompts, zero-shot vs role-based prompting, structured output with Pydantic, and your first LangGraph graph. | [clase1](./AI_Engineering/course1_prompt_engineering/clase1) |
| Class 2 — Prompt Chaining | Break a hard task into a chain of specialist nodes where the output of each step feeds the next. | [clase2](./AI_Engineering/course1_prompt_engineering/clase2) |
| Class 3 — Routing | Classify the intent and dispatch each case to the right specialist using conditional edges. | [clase3](./AI_Engineering/course1_prompt_engineering/clase3) |
| Class 4 — Evaluator-Optimizer | A generator/evaluator loop that improves the output with rubric-based feedback until it passes a measurable threshold. | [clase4](./AI_Engineering/course1_prompt_engineering/clase4) |
| Class 5 — ReAct | A tool-using agent that reasons and acts step by step, deciding which tool to call and when to stop. Includes a deep final challenge (`RETO.md`). | [clase5](./AI_Engineering/course1_prompt_engineering/clase5) |
| Bonus — Multi-LLM Comparison | Run the same flow with OpenAI and Gemini in parallel and let an LLM judge return a structured verdict (A/B testing with LLM-as-judge). | [comparativa_llms](./AI_Engineering/course1_prompt_engineering/comparativa_llms) |
